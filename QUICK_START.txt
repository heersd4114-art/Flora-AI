❓ QUICK ANSWERS:

Q: Why is first scan slow?
A: The local AI model (ResNet-50) loads on first use, then it's cached.

Q: What if phone can't find the app?
A: Both must be on SAME WiFi. Run VERIFY_SYSTEM.bat to check services.

Q: Does it work without WiFi?
A: No, WiFi is needed to connect phone to laptop API.

Q: What if Gemini API still has quota?
A: It tries Gemini first (high accuracy), then falls back to local (medium accuracy).

Q: Can I use cellular instead of WiFi?
A: Only if you set up a public tunnel (more complex). WiFi is easiest.

🔧 TROUBLESHOOTING:

Problem: "Waiting for API..." on phone
Solution: 
  1. Run SETUP_WIFI_MODE.bat
  2. Wait 60 seconds
  3. Restart app on phone

Problem: "Service unreachable"
Solution:
  1. Run VERIFY_SYSTEM.bat
  2. Check WiFi connection
  3. Make sure both devices on same network

Problem: Slow performance
Solution:
  1. First scan loads local AI (normal: 30-60s)
  2. Next scans cached (normal: 5-10s)
  3. Check WiFi signal strength
